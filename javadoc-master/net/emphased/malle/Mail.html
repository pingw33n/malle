    public interface Mail
    Fluent mail builder. This is the main interface to use when composing and sending a mail message.

    Depending on whether it's multipart or not (i.e. created with mail(true) or mail(false)) certain methods will throw UnsupportedOperationException. A non-multipart message doesn't support:

    • Having both plain and HTML bodies within a single message. See plain(String), html(String), body(BodyType, String).
    • Attachments (regular and inline). See attachment(InputStreamSupplier, String, String), inline(InputStreamSupplier, String, String) (and other variants of these methods).

    Minimal usage example: 

     MailSystem mailSystem = ... ;

 mailSystem.mail()
           .from("me@example.com")
           .to("you@example.com")
           .subject("Hello there")
           .html("<p>Hey hey hey</p>")
           .send();

      • address

        Mail address(AddressType type,
           String addresses)
        Adds the specified addresses to the address header specified by type. If needed the personal parts will be encoded according to the RFC822 using the charset.

        Some examples: 

             .address(AddressType.FROM, "from@example.com")
     .address(AddressType.TO, "John Doe <to@example.com>")
     .address(AddressType.CC, "♡ Unicode ♡ <cc1@example.com>, cc2@example.com")
     .address(AddressType.CC, "cc1@example.com")
     .address(AddressType.CC, "cc2@example.com")
        Parameters:
        addresses - one or more addresses encoded according to RFC822. Unicode personal part doesn't have to be encoded (see examples above).

      • address

        Mail address(AddressType type,
           String address,
           @Nullable
           String personal)
        Adds a single address to the address header specified by type. An optional personal may be specified and if needed it will be encoded according to the RFC822 using the charset.

        Note that this method is different from address(AddressType, String) in that the address must be a single local@domain address without the personal part (it may be specified as the personal parameter).

      • template

        Mail template(String name,
            @Nullable
            Locale locale,
            Object... context)
        A version of template(String, Locale, Map) with the context passed as varargs parameter.

        The following code 

             Mail m = ...;
     m.template("template name", null,
                "key1", "value1",
                "key2", "value2");
        does the same as 
             Map<String, Object> context = new LinkedHashMap<>();
     context.put("key1", "value1");
     context.put("key2", "value2");

     Mail m = ...;
     m.template("template name", null, context);
        Parameters:
        context - a flattened map in a form of a (key, value) pair sequence
        Throws:
        IllegalArgumentException - if the length of the the context array is not even
        IllegalArgumentException - if a key element in the context array is not a String
